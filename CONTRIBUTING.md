# Contributing to TapTapTidy

Thanks for your interest in contributing. TapTapTidy is a chore-tracking application, and contributions of all kinds are welcome.

## Ways to Contribute

- **Bug reports** — describe what happened, what you expected, and how to reproduce it
- **Feature ideas** — explain the problem and why the change would help
- **Documentation** — improve README, guides, or comments
- **Code** — fixes and features via pull requests

## Getting Started

1. Fork the repository and create a branch from `main`
2. Make your changes with clear, focused commits
3. Open a pull request with a short summary and, if relevant, test notes

## Pull Requests

- Keep changes scoped to a single concern when possible
- Follow existing code style and patterns in the project
- Update documentation if behavior or setup changes
- Ensure the project builds and tests pass before requesting review (once a build/test workflow exists)

## Issues

Search [existing issues](https://github.com/matzekFloyd/TapTapTidy/issues) before opening a new one. For bugs, include:

- Environment (OS, browser, app version, etc.)
- Steps to reproduce
- Expected vs. actual behavior
- Screenshots or logs, if helpful

## Development

Setup instructions will be added to the README as the project grows. If something is missing or unclear, open an issue or ask in your pull request.

## Code of Conduct

Be respectful and constructive. Harassment or abusive behavior is not tolerated.
